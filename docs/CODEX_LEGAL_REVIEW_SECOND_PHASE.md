# Codex 법령검토 2차 개선 계획

이 문서는 `korean-law-mcp`를 Codex에서 사용할 때 법령 검토 품질을 추가로 높이기 위한 2차 개선 기준이다. 목적은 단순 법령 검색을 넘어, 공공기관 실무 검토보고서 수준의 결론·근거·리스크·개선조치까지 자동 산출하는 것이다.

---

## 1. 개선 목표

Codex가 법령 검토 요청을 받을 때 다음 수준의 결과물을 안정적으로 작성하도록 한다.

| 목표 | 설명 |
|---|---|
| 조문 정확성 | 법령명, 조·항·호·목, 시행일을 최대한 특정 |
| 위계 검토 | 법률 → 시행령 → 시행규칙 → 고시·훈령·예규 → 조례·내규 순서 확인 |
| 사안 적용 | 조문 요건과 사용자가 제시한 사실관계를 1:1로 매칭 |
| 반대논리 검증 | 감사자, 법률자문, 상대방이 제기할 수 있는 쟁점 검토 |
| 실무 산출 | 보고서, 체크리스트, 증빙목록, 개선조치까지 생성 |
| 불확실성 통제 | 확인되지 않은 판례·해석례·조문은 임의 작성 금지 |

## 2. 우선 개발 대상 체인 도구

### 2.1 `chain_public_agency_review`

공공기관 일반 법령검토 전용 체인 도구.

#### 목적

지방공기업, 지방공단, 공공기관이 수행하는 시설관리·계약·안전·예산·공유재산 업무의 법적 적정성을 검토한다.

#### 입력 예시

```json
{
  "query": "공공체육시설 내 동호회가 설치한 불법 가설물과 LPG 사용에 대한 공단의 철거·사용중지 조치 가능 여부",
  "agencyType": "지방공기업",
  "issueType": "시설관리/안전/공유재산",
  "outputMode": "report"
}
```

#### 내부 실행 흐름

1. `search_ai_law`로 관련 법령 후보 탐색
2. `search_all`로 법령·행정규칙·자치법규 병렬 검색
3. `get_law_text`로 핵심 조문 원문 확인
4. `get_three_tier`로 위임관계 확인
5. `search_interpretations`, `search_precedents`, `search_admin_appeals`로 해석·사례 확인
6. 결과를 공공기관 법령검토 보고서 형식으로 정리

#### 출력 구조

```markdown
## 1. 검토 결론
## 2. 사실관계
## 3. 관련 법령
## 4. 조문별 요건 매칭
## 5. 반대논리 및 감사 리스크
## 6. 개선조치
## 7. 필요 증빙자료
```

---

### 2.2 `chain_contract_review`

지방계약·공유재산·예산절차 검토 전용 체인 도구.

#### 적용 대상

- 수의계약 가능 여부
- 협상에 의한 계약 가능 여부
- 위탁·사용허가·관리위탁 구분
- 계속비·채무부담행위·예산 외 의무부담
- 설계변경, 선금, 하도급, 품질·안전관리비 집행
- 물품·용역·공사 계약 적정성

#### 우선 검색 법령군

- 지방자치단체를 당사자로 하는 계약에 관한 법률
- 지방계약법 시행령·시행규칙
- 지방자치단체 입찰 및 계약집행기준
- 지방재정법
- 지방회계법
- 공유재산 및 물품 관리법
- 지방공기업법

#### 출력 판단값

| 판단값 | 의미 |
|---|---|
| 적합 | 조문 요건 충족 및 주요 리스크 낮음 |
| 조건부 가능 | 가능하나 의결·공고·심사·증빙 등 선행조건 필요 |
| 부적합 | 법령상 제한 또는 절차상 중대한 하자 존재 |
| 추가확인 필요 | 사실관계·예산·계약서·고시 등 확인 필요 |

### 2.3 `chain_safety_review`

산업안전보건·중대재해·시설물 안전 검토 전용 체인 도구.

#### 적용 대상

- 위험성평가 적정성
- 도급사업 안전보건관리
- 작업환경측정·특수건강진단
- 화학물질·온열질환·밀폐공간·추락·감전 위험
- 장사시설, 체육시설, 공동구, 하수처리시설 등 복합시설 안전관리
- 중대재해처벌법상 경영책임자 의무 이행 여부

#### 우선 검색 법령군

- 산업안전보건법
- 산업안전보건기준에 관한 규칙
- 중대재해 처벌 등에 관한 법률
- 시설물의 안전 및 유지관리에 관한 특별법
- 재난 및 안전관리 기본법
- 소방시설 설치 및 관리에 관한 법률
- 전기안전관리법
- 승강기 안전관리법

#### 출력 구조

| 구분 | 내용 |
|---|---|
| 위험요인 | 유해·위험요인 목록 |
| 법적 의무 | 법령상 의무조항 |
| 현재 수준 | 사용자가 제공한 사실관계 기준 판단 |
| 미흡사항 | 법령·지침 대비 부족한 사항 |
| 개선조치 | 즉시·단기·중장기 조치 |
| 증빙 | 점검표, 사진, 교육자료, 회의록, 계약서 등 |

### 2.4 `legal_review_report`

검색 결과를 보고서 형식으로 자동 변환하는 후처리 도구.

#### 목적

여러 도구에서 나온 검색 결과를 공공기관 내부보고서로 바로 활용 가능한 형태로 정리한다.

#### 입력값

```json
{
  "facts": "사용자가 제공한 사실관계",
  "laws": ["확인된 법령·조문"],
  "cases": ["해석례·판례·행정심판례"],
  "riskLevel": "A/B/C",
  "outputStyle": "executive_report"
}
```

#### 출력값

- 요약 5줄
- 쟁점별 결론표
- 법령·사실관계 매칭표
- 감사 리스크표
- 개선조치표
- 증빙자료 요청목록

---

### 2.5 `audit_risk_matrix`

감사 대응용 위험도 매핑 도구.

#### 판단 기준

| 등급 | 기준 |
|---|---|
| A | 법령 위반 가능성 높음, 감사 지적·제재 가능성 큼 |
| B | 절차 또는 증빙 미흡, 보완 시 리스크 완화 가능 |
| C | 법령 위반 가능성 낮으나 내부관리 개선 필요 |

#### 출력 예시

| 쟁점 | 법령 위반 가능성 | 감사 리스크 | 보완자료 | 조치기한 |
|---|---|---|---|---|
| 수의계약 사유 | B | 계약심사 지적 가능 | 수의계약 사유서, 시장조사서 | 계약 전 |
| 도급 안전관리 | A | 중대재해·산안법 지적 가능 | 위험성평가, 협의체 회의록 | 즉시 |

## 3. Codex 자동 적용 프롬프트

Codex가 법령 검토 요청을 받을 때 사용자가 별도 지시를 하지 않아도 아래 원칙을 적용한다.

```text
법령 검토 요청으로 판단되는 경우 AGENTS.md와 docs/CODEX_LEGAL_REVIEW_SECOND_PHASE.md를 우선 적용한다.

다음 순서로 처리한다.
1. 사실관계와 추가확인 필요사항을 분리한다.
2. 관련 법령 후보를 검색하고 법률·시행령·시행규칙·고시·조례 위계를 구분한다.
3. 조문 원문을 확인하고 조문 요건과 사실관계를 매칭한다.
4. 해석례·판례·행정심판례·감사 유사사례가 필요한 경우 추가 검색한다.
5. 결론은 적합 / 부적합 / 조건부 가능 / 추가확인 필요로 표시한다.
6. 반대논리, 감사 리스크, 보완조치, 증빙자료를 반드시 포함한다.
7. 확인되지 않은 내용은 추정하지 않는다.
```

## 5. 테스트 케이스 후보

아래 사례로 Codex 품질을 비교 검증한다.

1. ESCO 사업자파이낸싱 계약의 예산 외 의무부담 여부
2. 공공체육시설 내 동호회 화기 사용 제한 가능 여부
3. 은하수공원 장사시설 안전보건관리 의무 범위
4. 관리위탁과 사용수익허가 구분
5. 수의계약 가능 여부 및 감사 리스크
6. 도급사업 안전보건관리비 집행 적정성
7. 온열질환 예방조치 법적 의무
8. 화학물질 위험성평가 누락 시 개선조치
9. 시설물 정기안전점검 대가 산정 적정성
10. 공사 설계변경 및 품질관리비 중복계상 여부

## 6. 품질 검증 체크리스트

Codex 결과물은 아래 항목을 기준으로 검토한다.

| 점검항목 | 충족 여부 |
|---|---|
| 법령명과 조문이 특정되었는가 |
| 시행령·시행규칙 등 하위법령을 확인했는가 |
| 고시·훈령·예규·조례 필요성을 검토했는가 |
| 조문 요건과 사실관계를 매칭했는가 |
| 반대논리를 제시했는가 |
| 감사 리스크를 구분했는가 |
| 보완조치가 실행 가능하게 제시되었는가 |
| 필요한 증빙자료가 정리되었는가 |
| 불확실한 부분을 명확히 표시했는가 |
| 허위 판례·해석례·조문 추정이 없는가 |

---

## 7. 결론

2차 개선의 핵심은 “도구 추가”보다 “검토 흐름의 표준화”이다. Codex가 법령 검토 요청을 받으면 AGENTS.md의 기본 절차와 본 문서의 2차 개선 기준을 함께 적용하여, 단순 검색 요약이 아니라 공공기관 보고서 수준의 검토 결과를 산출해야 한다.
