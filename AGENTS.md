# AGENTS.md

Korean Law MCP Server v2.1 — Codex 작업 지침

이 문서는 Codex가 `korean-law-mcp` 저장소에서 작업할 때 반드시 따라야 할 프로젝트 규칙과 법령검토 품질 기준이다. Claude용 `CLAUDE.md`와 동일한 목적을 가지며, Codex가 법령 검토 요청을 단순 검색 결과 요약으로 끝내지 않고, 근거 중심의 검토보고서 수준으로 산출하도록 강제한다.

---

## 1. 프로젝트 개요

- 이 저장소는 법제처 Open API 기반의 한국 법령 MCP 서버이다.
- 대상 정보는 법령, 시행령, 시행규칙, 행정규칙, 자치법규, 판례, 법령해석례, 행정심판례, 헌재결정례, 위원회 결정례, 별표·서식, 법령용어 등이다.
- MCP 서버와 CLI를 모두 지원하며, 자연어 질의를 내부 도구 호출로 변환한다.
- 법령 검토 품질은 “검색 여부”가 아니라 “정확한 조문, 시행일, 위임관계, 해석례·판례, 사안 적용, 반대논리 검토”로 판단한다.

## 2. 기본 작업 명령

```bash
npm install
npm run build
npm run watch
LAW_OC=키 node build/index.js
```

CLI 사용 예시:

```bash
korean-law "지방계약법 수의계약 가능 여부"
korean-law "산업안전보건법 온열질환 예방 의무"
korean-law search_law --query "지방자치단체를 당사자로 하는 계약에 관한 법률"
korean-law get_law_text --mst 279811 --jo "제9조"
```

## 3. 코드 수정 원칙

1. `src/tools/*.ts`의 모든 입력값은 Zod 스키마로 검증한다.
2. 새 도구를 추가하면 반드시 `src/tool-registry.ts`의 `allTools`에 등록한다.
3. STDIO 모드 간섭 방지를 위해 `console.log`와 `console.error`를 사용하지 않는다. 오류는 구조화된 에러 또는 throw로 처리한다.
4. API 응답 배열 필드는 단일 객체로 올 수 있으므로 반드시 `Array.isArray(x) ? x : [x]` 패턴으로 정규화한다.
5. 모든 최종 도구 출력에는 응답 크기 제한을 적용한다.
6. `search-normalizer.ts`, `law-parser.ts` 등 LexDiff 기반 코드는 원본 확인 없이 임의 수정하지 않는다.
7. 조문번호는 JO 코드 변환 오류가 잦으므로 `parse_jo_code` 또는 기존 파서 로직을 우선 사용한다.
8. 빌드가 가능한 변경인지 반드시 `npm run build`로 검증한다.

## 4. 법령 검토 시 Codex 응답 절차

법령 검토 요청을 받으면 아래 순서를 지켜야 한다. 한 번의 자연어 답변으로 추정하지 말고, 가능한 경우 MCP 도구를 사용해 근거를 확보한다.

### 4.1 사실관계 정리

- 검토 대상 기관, 사업, 행위, 계약, 시설, 사고, 안전조치, 예산·회계 절차를 구분한다.
- 사용자가 제공한 사실과 추가 확인이 필요한 사실을 분리한다.
- 불확실한 부분은 결론을 단정하지 않고 “추가확인 필요”로 표시한다.

### 4.2 법령 검색

우선순위:

1. `search_ai_law` 또는 `search_all`로 관련 법령 후보 탐색
2. `search_law`로 정확한 법률명 확정
3. `get_law_text` 또는 `get_batch_articles`로 조문 원문 확인
4. `get_three_tier` 또는 `get_law_tree`로 법률→시행령→시행규칙 위임관계 확인
5. `get_annexes`로 별표·서식·기준이 있는지 확인
6. 필요 시 `search_admin_rule`, `search_ordinance`로 고시·훈령·예규·조례 확인

### 4.3 해석·사례 검색

쟁점이 있는 경우 다음을 추가 확인한다.

1. `search_interpretations` 및 `get_interpretation_text`
2. `search_precedents` 및 `get_precedent_text`
3. `search_admin_appeals` 및 `get_admin_appeal_text`
4. 사안별로 조세심판, 관세, 공정위, 노동위, 개인정보위, 헌재 결정례 검색

### 4.4 판단 구조

최종 판단은 다음 구조를 따른다.

| 구분 | 작성 기준 |
|---|---|
| 결론 | 적합 / 부적합 / 조건부 가능 / 추가확인 필요 |
| 근거 조문 | 법령명, 조문, 항·호·목, 시행일 |
| 판단 이유 | 조문 요건과 사실관계의 1:1 매칭 |
| 반대논리 | 상대방 또는 감사자가 제기할 수 있는 쟁점 |
| 리스크 | 감사, 계약, 예산, 안전, 민원, 소송 위험 |
| 보완조치 | 즉시 / 단기 / 중장기 조치 |
| 증빙자료 | 결재문서, 계약서, 산출내역, 회의록, 사진, 점검표 등 |

## 5. 법령 검토 보고서 표준 출력

법령 검토 산출물은 다음 순서를 기본으로 한다.

```markdown

## 1. 검토 요약
- 결론:
- 핵심 근거:
- 주요 리스크:
- 즉시 조치:

## 2. 사실관계
| 항목 | 내용 | 확인 수준 |
|---|---|---|

## 3. 쟁점 정리
| 번호 | 쟁점 | 검토 필요성 |
|---|---|---|

## 4. 관련 법령 및 기준
| 구분 | 법령·기준 | 조문 | 주요 내용 | 사안 적용 |
|---|---|---|---|---|

## 5. 조문별 판단
| 쟁점 | 요건 | 사실관계 | 판단 | 리스크 |
|---|---|---|---|---|

## 6. 해석례·판례·감사 유사사례
| 출처 | 사건·문서 | 핵심 내용 | 적용 가능성 |
|---|---|---|---|

## 7. 종합 판단
- 적합/부적합/조건부 가능/추가확인 필요 중 하나로 명시한다.
- 단정하기 어려운 경우 이유와 추가 확인자료를 명시한다.

## 8. 개선조치
| 구분 | 조치사항 | 담당 | 기한 | 증빙 |
|---|---|---|---|---|
```

## 6. 한국 공공기관 검토 시 우선 고려 법령군

공공시설, 지방공기업, 계약, 안전, 시설관리 검토에서는 아래 법령군을 우선 고려한다.

- 지방공기업법
- 지방자치법
- 지방재정법
- 지방회계법
- 지방자치단체를 당사자로 하는 계약에 관한 법률 및 시행령·시행규칙
- 공유재산 및 물품 관리법
- 산업안전보건법 및 산업안전보건기준에 관한 규칙
- 중대재해 처벌 등에 관한 법률
- 시설물의 안전 및 유지관리에 관한 특별법
- 재난 및 안전관리 기본법
- 소방시설 설치 및 관리에 관한 법률
- 건설기술 진흥법
- 전기안전관리법
- 승강기 안전관리법
- 에너지이용 합리화법
- 개인정보 보호법
- 장사 등에 관한 법률

관련 법령은 항상 사안에 맞게 추가 검색한다.

## 7. 품질 기준

Codex의 최종 답변은 다음 기준을 충족해야 한다.

1. 조문명만 나열하지 않고, 조문 요건과 사실관계를 연결한다.
2. 법령명, 조문, 항·호·목을 최대한 특정한다.
3. 현재 시행 중인지, 개정 전후 비교가 필요한지 확인한다.
4. 상위법, 시행령, 시행규칙, 고시, 조례의 위계관계를 구분한다.
5. 해석례·판례가 없으면 “확인된 자료 없음”이라고 쓴다.
6. 법률전문가 최종 검토가 필요한 사안은 “법률자문 필요”로 표시한다.
7. 사용자가 공단 보고서로 활용할 수 있도록 행정보고서 문체와 표 형식을 우선한다.
8. 모르는 내용은 추정하지 않는다.

## 8. Codex 전용 실행 방식

복잡한 법령 검토는 한 번에 처리하지 말고 다음 3단계로 나누어 수행한다.

1. 조사 단계: 관련 법령·조문·해석례·판례 후보 수집
2. 판단 단계: 사실관계와 조문 요건 매칭
3. 검증 단계: 반대논리, 감사 리스크, 추가 확인자료 검토

Codex가 직접 보고서를 작성할 때는 먼저 짧은 계획을 제시하고, 이후 근거표를 만든 다음 최종 보고서를 작성한다.

## 9. 금지 사항

- 법령 원문 확인 없이 결론부터 제시하지 않는다.
- “일반적으로 가능하다”와 같은 모호한 표현으로 끝내지 않는다.
- 조문번호를 임의 추정하지 않는다.
- 판례·해석례가 없는데 있는 것처럼 작성하지 않는다.
- 사용자가 제공하지 않은 사실관계를 단정하지 않는다.
- API 오류, 검색 실패, 응답 누락을 숨기지 않는다.

- “상위법상 근거는 있으나, 조례·내부규정·계약조건과의 정합성 검토가 필요합니다.”
